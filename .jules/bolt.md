## 2026-04-29 - Render loop and re-render optimization in Home/Card
**Learning:** In scenarios where a parent state (like a search input) changes frequently, heavy filtering and deduplication logic inside the render loop can significantly degrade performance. Moving this logic to 'useMemo' and using an O(N) Map-based grouping instead of O(N^2) nested filtering reduces data processing time from O(Categories * Items^2) to O(Items). Additionally, wrapping child list items in 'React.memo' prevents redundant re-renders of the entire list when only the parent's independent state changes.
**Action:** Always check for O(N^2) patterns in render loops, especially when they involve filtering/reducing large arrays. Use 'React.memo' for list items that receive stable props from a parent with frequent state updates.
